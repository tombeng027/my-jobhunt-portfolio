Portfolio Documentation

This page explains the intent, architecture, and change history of this portfolio. It serves as a quick reference for future updates and for employers who want to understand how the site is maintained.

Vision & Mission

This portfolio is designed to showcase implementation engineering, full-stack capability, and configuration-first delivery experience.

  • Vision: Present a modern, credible technical profile with clear project evidence.
  • Mission: Help hiring teams evaluate delivery strength quickly through an accessible live portfolio.

How It Was Built

  • Built with Vite (vanilla JS) for fast local development and lightweight production output.
  • Main portfolio uses a two-column v2 layout (sticky profile panel + section-based content flow).
  • Documentation page uses a dedicated static stylesheet (public/documentation.css) for reliable deployment.
  • Repository: github.com/tombeng027/my-jobhunt-portfolio.

How This Portfolio Was Updated

The portfolio has been iteratively refined to align with current job-hunt goals and improve technical presentation.

Recent update log

  • Created and published GitHub repository with GitHub Pages deployment workflow.
  • Fixed GitHub Pages issues (404, workflow location, action version, permissions, and Pages source setup).
  • Corrected production asset path handling for Vite by configuring relative base paths.
  • Integrated working project links for Repo and Docs in the portfolio project card.
  • Moved documentation into the public output path so it is deployable and accessible from the live site.
  • Replaced generic hero code sample with a configuration-focused snippet aligned to implementation work.
  • Created version-2 branch and redesigned the main site into a minimalist two-column structure inspired by modern developer portfolios.
  • Design direction was inspired by modern developer portfolio patterns, including brittanychiang.com, while keeping this implementation and content original to my profile.
  • Updated navigation behavior to highlight active sections and support mobile section toggling.
  • Redesigned this documentation page to match v2 styling and removed fragile hashed asset dependencies.
  • Created Old Software Showcase page to archive earlier projects from 2018–2020, with dedicated styling and project cards.
  • Added an E-Commerce Showcase to demonstrate full-stack thinking and UI/UX understanding — product listing, cart, and checkout flows implemented using local storage, with no external APIs at this stage.
  • Upgraded the E-Commerce Showcase to consume the FakeStore REST API — products and categories are now fetched live using fetch and async/await, replacing the previous hardcoded data. Includes loading and error states, dynamic category filter population, and real product images.
  • Added a localStorage cache fallback to the E-Commerce Showcase — API responses are saved on success and served from cache if the API is unavailable, ensuring the showcase remains functional offline or during API downtime.

Deployment

The site deploys automatically via GitHub Actions to GitHub Pages.

  1. Push changes to the tracked branch.
  2. Workflow builds the project with npm run build.
  3. Generated files are published to gh-pages.

Live URL: tombeng027.github.io/my-jobhunt-portfolio.
