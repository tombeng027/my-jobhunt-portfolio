Portfolio Documentation

This page explains why this portfolio exists, how it was built, and how to update it as you continue to grow.

Vision & Mission

This portfolio is designed to showcase full-stack development and infrastructure configuration experience in a clean, copyable format that employers can quickly evaluate.

  • Vision: Present a polished, modern technical portfolio that feels like a real product.
  • Mission: Help you land a web development role by highlighting technical depth (full-stack + config tooling) and delivering a deployable site.

How it was built

  • **Vite (vanilla JS)** for a fast dev workflow and production build.
  • **Static HTML + CSS** for predictable rendering and easy customization.
  • **Modular structure**: the site is a single-page portfolio with a second documentation page.
  • **Responsive design** with a mobile-friendly nav and clean sections.
  • The source code is available in the repository: github.com/tombeng027/my-jobhunt-portfolio.
  • The “Docs” button on the Portfolio Site project card links to this page.

How to update this portfolio

  1. Edit index.html to change the headline, sections, project cards, or contact details.
  2. Update src/style.css to adjust styles, colors, and layout.
  3. To add a new project card, copy an existing <article class="project"> block and update the content and links.
  4. To add more documentation pages, create a new HTML file at the root of portfolio-site/ and link to it from the navigation.

Deploying this portfolio

The repository can be deployed to any static hosting provider. The quickest options are:

  • Vercel – connect the repo and deploy automatically
  • Netlify – drag-and-drop dist/ or connect via repo
  • GitHub Pages – build to dist/ and publish the output

Run npm run build to generate the production output in dist/.
